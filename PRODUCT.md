# Product

## Register

product

## Users

同事和朋友之间需要跨设备传输文件或文本。使用场景包括局域网内快速交换，以及公网下的点对点直连。用户希望打开即用，不需要注册账号或学习复杂的操作流程。

## Product Purpose

让任何两台设备之间能以最短路径传输文件或文本。没有中间服务器存储，没有账号体系，没有多余的步骤。成功就是用户选中文件、发送、对方收到，三步完成。

## Brand Personality

快速、极简、可靠。三个词贯穿 UI 和体验的每一个细节。

- **快速**：操作路径最短，响应无需等待。传输进度对用户透明。
- **极简**：不展示用户不需要的东西。每个屏幕只有一个核心操作，没有冗余控件。
- **可靠**：状态明确，连接和传输状态始终可见，错误可理解。

## Anti-references

- 不要赛博朋克风格（霓虹、故障效果、暗黑过载）
- 不要花里胡哨（装饰性元素、不必要的动画、过度设计）
- 不要像 Telegram（功能密集、侧栏多层、视觉过载）
- 不要复杂（不到三个步骤能完成的事，就不要做成配置向导）

## Design Principles

1. **一条主路径** — 每个页面只提供一个核心操作。加入房间 → 传输，两个步骤，不分支。
2. **状态透明** — 用户永远知道当前处于什么阶段（连接中、传输中、完成），不需要猜测。
3. **工具的克制** — UI 退后，内容前进。不要为了设计而设计，每个像素都要服务于传输这个目标。
4. **容错优先** — 连接失败、粘贴无效、传输中断……异常状态和正常状态一样被设计对待。

## Accessibility & Inclusion

标准 WCAG 级别。足够的色彩对比度，支持键盘操作，所有交互元素有焦点态。减少不必要动画，对 motion 偏好有基本尊重。
