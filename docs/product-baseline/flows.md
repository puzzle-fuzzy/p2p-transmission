# 核心用户流程基线

## F1：创建房间

Given 用户位于大厅<br>
When 点击“创建房间”<br>
Then 显示房间码、分享入口、倒计时、发送方身份和接收者占位；未连接接收者时不能发送。

## F2：手动房间码加入

Given 发送方有有效房间，接收方只有 6 位房间码<br>
When 接收方提交加入请求<br>
Then 接收方进入等待；发送方收到申请；只有发送方允许后才成为房间成员。

拒绝、取消、房间不存在和请求过期都必须结束等待并给出下一步。

## F3：邀请链接加入

Given 接收方获得有效邀请链接<br>
When 打开链接并确认加入<br>
Then 使用链接中的 capability 完成授权，不把 capability 显示在普通日志或分享给其他房间。

当前实现可以重做链接格式，但必须保留“邀请链接有权限、房间码只有定位作用”的安全含义。

## F4：接收者上线

Given 发送方房间已打开<br>
When 一个批准的接收者实时连接<br>
Then 接收者头像从小到大出现，连接人数和 aria-live 状态更新；reduced-motion 下无缩放动画。

## F5：发送单个文件

Given 双方 WebRTC DataChannel 已连接<br>
When 发送方选择文件并点击发送，接收方接受<br>
Then 双方看到进度；接收完成后可下载；发送方看到该接收者完成。

## F6：粘贴文本

Given 发送方焦点位于上传区域<br>
When 粘贴文本并确认<br>
Then 文本作为 `粘贴内容.txt` 加入队列；接收完成后既可下载也可复制内容。

## F7：定向与广播

Given 两位或更多接收者在线<br>
When 发送方只选择其中一位<br>
Then 只有被选择者收到请求；再次选择全部后，所有被选择者各自收到请求并独立返回结果。

## F8：取消、拒绝与重试

- 接收方拒绝只影响自己的 outcome。
- 任一方取消后停止读写并释放当前传输资源。
- 一个接收者失败不能错误终止其他接收者。
- “再次发送”创建新一轮传输，同时保留上一轮可理解的结果摘要。

## F9：刷新与断线

- 页面刷新或短暂网络中断后，不显示陈旧的“已连接”。
- 可以恢复时给出恢复入口；不能恢复时明确返回大厅或重新加入。
- 所有后台旧任务不得回写新房间状态。
