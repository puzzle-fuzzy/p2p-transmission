# 本地开发连接修复与 Toast 布局设计

日期：2026-07-14

状态：已确认，待实施

## 背景

本地启动项目时，页面会显示“暂时无法连接服务器”。排查发现本项目 API 默认使用 `3000` 端口，但本机已有 `X:\enshrine` 进程占用该端口。前端默认请求 `http://localhost:3000`，因此会命中其他服务，`/v1/visitors` 返回 404，访客初始化失败。

同时，现有全局 Toast 位于顶部居中，使用紫色描边，宽度和内边距偏大。用户希望消息移动到右上角，去除紫色 border，并稍微收紧尺寸比例。

## 目标

1. 让本项目在默认配置下避开本机常见的 `3000` 端口冲突。
2. 让前端默认 API 地址与 API 默认端口保持一致，同时保留环境变量覆盖能力。
3. 将全局 Toast 固定到右上角，保持移动端不溢出。
4. 移除 Toast 的紫色描边，保留不同提示级别的文字和图标语义。
5. 适度缩小 Toast 的最大宽度、内边距与整体视觉比例，同时保留关闭按钮的可操作尺寸。
6. 不改变生产环境显式配置端口的行为。

## 非目标

- 不修改 `X:\enshrine` 或其他项目的端口配置。
- 不自动终止占用 `3000` 的进程。
- 不改变 Toast 的消息内容、自动消失时间、错误/成功/普通三种 tone 语义。
- 不把生产环境 API 端口强制改成 `3332`；生产部署继续由环境变量决定。

## 方案

### 本地 API 端口

- 将 API 配置中的默认端口从 `3000` 改为 `3332`。
- 将 Web 客户端没有 `VITE_API_URL` 时的默认地址改为 `http://localhost:3332`。
- 同步更新 Web 和 API 的 `.env.example`，让示例配置与测试配置一致。
- 生产环境如果设置了 `PORT` 和 `VITE_API_URL`，仍使用显式值，不受默认值调整影响。

选择 `3332` 是因为项目的 Playwright E2E 已经使用该端口，且它与当前本机冲突的 `3000` 隔离；前端开发端口 `5713` 保持不变。

### Toast 位置与比例

- 外层定位由顶部居中改为 `top/right` 固定定位，在桌面端使用较小的安全边距，在窄屏保持左右留白。
- 最大宽度由 `360px` 收紧到约 `320px`，避免长消息横向占用过多空间。
- 去除容器的 border class，不再使用 `border-accent` 或其他彩色描边。
- 将容器内边距、间距和最小高度略微收紧；关闭按钮继续保持至少 44px 的触控区域。
- 保留当前 `role`、`aria-live`、`aria-atomic`、关闭按钮和 tone 图标行为。
- 保留现有进入动画，并确保窄屏消息不会超出视口。

## 验证标准

- 在没有 `VITE_API_URL` 的本地 Web 开发环境中，访客初始化请求发往 `http://localhost:3332/v1/visitors`。
- API 未设置 `PORT` 时监听 `3332`，设置 `PORT` 时仍监听设置值。
- Toast 的 DOM 定位包含右侧固定定位，不再包含紫色 border class。
- Toast 的最大宽度小于当前 `360px`，且在移动视口内不溢出。
- Web/API 相关单元测试、lint、typecheck、build 通过；必要时用 Bun 进行本地端口探测。
